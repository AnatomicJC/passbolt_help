    View one resource

    It is possible to get data for a single resource identified by the unique UUID. All you need is to make a GET request to /resource/.json.

    GET /resource/<resourceId>.json?api-version=v2

    Possible responses

    Code Description
    200 OK
    Response includes the resource metadata object.
    400 Bad Request
    The resourceId supplied is not a valid UUID
    403 Authentication Failure
    The user making the request is not authenticated
    404 Not Found
    The resource either does not exist or the user does not have access permission.
